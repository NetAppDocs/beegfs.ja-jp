= BeeGFS CSI ドライバーを展開するための前提条件
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
BeeGFS CSI ドライバーをデプロイして構成する前に、Kubernetes クラスターが実行されていること、BeeGFS クラスターにアクセスできること、クラスター全体のリソースをインストールするための適切な権限があることを確認してください。



== Kubernetesのセットアップを確認する

Kubernetes クラスター内のポッドのステータスを確認するには、次のコマンドを実行します。

[source, console]
----
kubectl get pods
----
**期待される出力:**

[listing]
----
root@node01:/etc/cni/net.d# kubectl get pods
No resources found in default namespace.
----
実行中のポッドがない場合、リストは空になります。それ以外の場合、ポッドはエラーなしでリストされるはずです。



== BeeGFSクライアントとサービスの検証

次のコマンドを実行して、BeeGFS クラスターが正常であり、すべてのサービスにアクセスできることを確認します。

**BeeGFS 7.4**

[source, console]
----
beegfs-check-servers
----
**ビーGFS 8**

[source, console]
----
beegfs health check
----

NOTE: CSIドライバーは、 `beegfs-dkms-client` そして `beegfs-client`各 Kubernetes ノードにインストールする必要があります。インストール手順については、BeeGFS クライアント DKMS ドキュメントを参照してください。 https://doc.beegfs.io/latest/advanced_topics/client_dkms.html[]
